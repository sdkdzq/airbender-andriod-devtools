Release Notes

Version 10.03

  • The OpenMAX AL extension samples have been removed from the pack. MediaPlayer, MediaExtractor and MediaCodec samples covering similar use cases are planned for an upcoming release.

  • All of the samples have been upgraded to target SDK 15. Note that they will still run on older OS versions. The minimum SDK levels have all been set appropriately. The target SDK levels ensure a good experience on the latest Jellybean devices

  • The samples and templates that used the menu button have been modified to use on-screen buttons, as Android is phasing out the use of the menu button in modern applications.

Version 10.01
  • A bug in NvEGLUtil that caused it to fail the eglMakeCurrent call has been fixed. This call was not used in any of the samples pack samples or libraries, but may have affected apps using the library.
Version 10.00

Initial release.

Known Issues and Warnings

Note the following issues and warnings with respect to this version of the samples pack:

  • This is a new set of samples (and has not been battle-tested to the same degree as previous samples packs); applications late in the production process should use the code in this pack only as a reference for the features demonstrated. NVIDIA does not recommend all games move immediately to this framework.
  • Performance and latency of the event handling code in the new samples pack has not been profiled. This is being analyzed and tuned for an upcoming release. In addition, frame rate limiters are also being considered for the apps. This coming analysis may cause some of the event-queue handling in the sample apps’ main.cpp (and thus developer apps based on it) to need changes.
  • Several applications present in previous samples packs are not included or replaced by direct analogues in this new pack; specifically the shader-heavy “water”, the vertex-heavy “Lorenz” and the Java-based SurfaceTexture sample. Applications similar to these (and others) are slated to be added for later releases.
  • The user interfaces of some of samples and demos are currently rendering off of the edges of smartphone screens in portrait orientation. All are still usable, but this visual anomaly will be fixed in an upcoming release.
  • When exiting all of the samples on some ICS OS images, an Android warning/error is shown in logcat. This warning has not been root-caused, but note that the Google-supplied NativeActivity sample in the official NDK causes the same message:
