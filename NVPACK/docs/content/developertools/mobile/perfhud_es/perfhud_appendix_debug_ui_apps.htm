Appendix D: Debug UI Applications with PerfHUD ES

Most UI applications are different from game applications, because they draw frames on demand. However, PerfHUD ES is driven by drawing events. If PerfHUD ES connects to a UI application, you will not see data moving on the dashboard, or even be able to set up the initial connection.

As a result, in order to debug UI applications with PerfHUD ES, you need to do the following:

  • Try to produce drawing events by user actions. For example, scroll the view up or down on the device.
  • If you are the developer of the application, redraw the current frame repeatedly, even if there is no change on demand.
