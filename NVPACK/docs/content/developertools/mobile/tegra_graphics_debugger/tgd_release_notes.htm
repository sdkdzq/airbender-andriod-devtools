Release Notes

Tegra Graphics Debugger 2.0

What's New: 

  • Support for OpenGL 4.5.
  • Support for GL_ARB_buffer_storage from OpenGL 4.4.
  • Support for serialized captures via source code generation.
  • Added Shader Perf statistics in the Linked Programs view.
  • Support for SHIELD Console.
  • Support for launching on all Tegra X1 and above devices. (Note that Nexus 9 will require an OS update.) 
  • Added support for scrubbing through the target view on the tablet and SHIELD Console.

Fixed Issues: 

  • Global Interposer has been removed. (Note that this does not apply to the Nexus 9.) 
  • Many bug fixes.

Known Issues: 

  • In order to see the buffer selected in the Current Target View also on device you must have the HUD enabled. If you disable the HUD the rendering on target will revert to color target 0. (37274)
  • If you switch between the original and edited shaders multiple times, it is possible for the Current Target View to be out of sync. This will typically resolve of you switch back and forth again. (36757) 
  • If your application uses binary programs, the resulting serialized capture is only compatible with the same target device and OS version. The driver can embed information into the binary that locks it to the current system. To work around this, delete any shader cache before running the tool if you want to serialize the capture. (37163)
  • Applications that use glDrawTransformFeedback will not see correct profiling data for those calls. Other draw calls in the frame should be correct. (35563)
  • If you are seeing periodic failures connecting to a launched application, it could be because of a conflicting version of adb running on your host system. These can sometimes come from phone and tablet sync applets that run in the background, waking up periodically to check if a device is attached. Since they have their own (potentially older) version of adb in the install package, it can conflict with a newer one running from the NDK, installed by TADP and Tegra Graphics Debugger. Uninstalling the sync app should fix the conflict.
  • If you encounter an issue where you are not able to capture, you should use the adb logcat command to see if there are any warning messages. The types of messages commonly seen are: 
    1. Unsupported functions (see Supported OpenGL Functions for a complete list).
    2. Use of OpenGL ES 1.0 (which is not supported).
    3. Out of memory issues. (31527) 
  • The "Disable Vertical Sync" option in the directed tests panel was removed because it wasn't functioning properly. This should return in a future version. (31672) 
  • After docking the Profiler View, the "Weight by" combo box may not open properly. If you change to a different graph, say the Counters List, and back to the Summary it should work correctly again. (30254)
  • If your application is exhibiting strange behavior or rendering, or even crashes when launched with Tegra Graphics Debugger, you can try and fall back to the Global Interposer by clicking on the Attach tab in the Connect, and pressing the button in the upper right corner to enable the Global Interposer. Then, launch the app from the device by tapping on it, refresh the list of applications to attach to, and double-click the correct one that comes up. (31745) 
  • When using a target running Linux that uses ssh instead of adb, Tegra Graphics Debugger is unable to connect to the target device as a "root" user. This is because ssh config has forbidden root password authentication. If you check the contents of the target root fs /etc/ssh/sshd_config, you will find the following line:  
    PermitRootLogin without-password
    As a workaround, change the line to:  
    PermitRootLogin yes
    Alternatively, you can log in with a different user account, and should not encounter this same error. (33646)
