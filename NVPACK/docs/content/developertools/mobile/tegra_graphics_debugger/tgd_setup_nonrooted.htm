Using the Debugger: Non-Rooted

This guide will give you a walkthrough of using Tegra Graphics Debugger with a non-rooted device.

Configuring Your Application

If you are using a non-rooted device, you will need to modify your application before it will work with Tegra Graphics Debugger.

  1. All of the files that are needed to use an application with Tegra Graphics Debugger on a non-rooted device are included with the install package.
    • For a 32-bit application running on KitKat or Lollipop use the files in [install]/target/android-kk-t124-a32.
    • For a 64-bit application running on Lollipop use the files in [install]/target/android-L-t132-a64
  2. You need to link the application against libNvidia_gfx_debugger_stub.a. Typically, applications link against EGL and GLESv2 by specifying -lEGL and -lGLESv2 on the link command line, to resolve the EGL and GLES symbols. Remove those and add –lNvidia_gfx_debugger_stub, so the symbols are resolved to the Tegra Graphics Debugger’s library. Make sure that the path containing libNvidia_gfx_debugger_stub.a is included in your library path.

    Note that you do not want to have libNvidia_gfx_Debugger.so in the same path, or the application may link against the .so file and that will not work correctly.

  3. Modify your packaging step to include libNvidia_gfx_debugger.so and libNvPmApi.Core.so.
    Note that these have a prefix of Stripped_ in the name, so you will want to rename it to remove that prefix before using it in your APK.
  4. After you have modified the APK, deploy it to your device with the adb install command.

Attaching to Applications with Tegra Graphics Debugger

  1. Once you have deployed your application to your device, launch it by tapping the application icon. Select the tab on the Connection screen that says Attach, and click the refresh button just above the Application Name list to see the application of interest.

  2. Applications that have been configured for Frame Debugging will appear in the list.
    1. You can choose Show All if you want to view all applications that are running on the target device; applications that cannot be debugged will appear with their names grayed out.
    2. To quickly locate a specific application by name, you can type the first few letters of its name, or its process ID in the Search field, and only matching applications will appear in the list.
  3. Select the application and click Attach in the lower right corner to start using the Tegra Graphics Debugger.
