Release Notes

Tegra System Profiler 2.4

Supported Platforms

  • Android
    • Android TV
    • NVIDIA SHIELD Tablet
    • NVIDIA SHIELD Portable (requires OTA OS update 65 or above)
    • Google Project Tango Tablet
    • Google Nexus 9 (requires OS v5.1 rooted)
    • Various devkits
  • L4T (Linux For Tegra)
    • Jetson TK1
    • Jetson TX1
  • Vibrante (with Android, Ubuntu, or Yocto)
    • Jetson Pro TK1

What's New

  • Timeline tooltips now show the thread blocked state call-stack when you hover the mouse over the thread’s blocked state segment. This will improve the user’s ability to diagnose the cause of a thread blocking.
  • Thread names are now auto-populated from OS APIs without NVTX. NVTX may override the name provided through the OS.
  • Threads may be sorted by utilization.
  • Support for CUDA API trace and tracing corresponding GPU workloads. L4T and Vibrante only.
  • NVTX (NVIDIA Tools Extensions) are now supported on 64-bit processes.
  • Various performance and backtrace quality improvements, bug fixes.

Known Issues

  • Profiling with max frequency for long periods of time may cause buffer overflow errors. (Issue #25660) 
  • Backtraces are sometimes incorrect, particularly for functions that take very little time. (If the sampling occurs during the prolog/epilog the PC register.) Sampling for a longer period of time and ignoring low-impact functions (hiding these often-wrong, seldom-occurring values) may help. (Issue #14669)
  • Profiling on Vibrante Yocto for longer than a minute may result in termination of the profiling session by system.
  • Profiling on L4T and Vibrante will require apt-get debug packages for OS libraries from Ubuntu and Yocto to successfully backtrace.
  • The following devices do not support L1/L2 counters: Google Project Tango Tablet, Google Nexus 9, and NVIDIA SHIELD Tablet prior to OTA 2.2.
  • Google Nexus 9 OS v5.1x and below require root privileges. This issue will be fixed in a future release.
