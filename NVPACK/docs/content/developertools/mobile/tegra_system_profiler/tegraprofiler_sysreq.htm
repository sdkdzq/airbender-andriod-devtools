System Requirements

To use Tegra System Profiler, you must first:

  1. Have a compatible Android device.
  2. Have a compatible Android OS image.

    NOTE: Devices from NVIDIA typically ship with support, while OEMs or ODMs producing their own Tegra device may choose whether or not to support this tool.

  3. Install the Android SDK Platform tools.
  4. Install the ADB USB Driver.
  5. Have your target application compiled with the correct flags and packaged with the correct permissions.

Items 3-4 can be handled automatically for you by installing the latest NVIDIA AndroidWorks. Item 2 may already be on your device, if it is a retail device starting with the Tegra 4 (minimum), but not all devices are required to come with OS images that support this tool. Item 2 can also be found for devkits by downloading the latest OS Image for your device. AndroidWorks can be found at:

https://developer.nvidia.com/androidworks

Using AndroidWorks is the recommended path for getting your device and system prepared to use Tegra System Profiler.

Compiling Your Application for Sampling

All NDK compiled applications are compiled with -funwind-table. If you are using NDK this is enough. Nsight Tegra 1.5 or above will also automatically add this gcc/clang flag for the user. If gcc/clang is used directly then -funwind-table should be added. This flag does not require exception handling to be enabled.

In order to keep the information needed for accurate sampling, your application needs to be compiled with the flags -fno-omit-frame-pointer and -marm. Additionally, in rare situations you may find incorrect backtraces near leaf functions, where one function will be skipped because maintaining frame pointers can be skipped for leaves, if the code was built with optimization. In the event that this is a problem, you may wish to build your code without optimization, for example: Application.mk: APP_OPTIM:=debug, though we recommend you try it without this first, as the rest of the timing data will be more accurate without it.

Manually Installing the Android SDK

If you choose not to use AndroidWorks, you need to download and install the Android SDK:

http://developer.android.com/sdk/index.html

Google has changed their setup slightly over time. The bundle is especially different. The location for ADB must be added to the PATH environment variable manually.

Manually Installing the USB Driver

NVIDIA AndroidWorks includes a USB driver. If you did not use AndroidWorks, or if your specific driver is missing for your NVIDIA device, please check NVIDIA’s devzone for a driver. You may also download the standard Google driver and force it to be installed, despite missing the matching USB PID/VID number for your device.

http://developer.android.com/sdk/win-usb.html

For instructions on installing the driver manually, see Getting Started with AndroidWorks.
