Release Notes

Tegra Graphics Debugger 2.6

What's New:

  • Android O support
  • Vulkan 1.0 support

    Note the following limitations with Vulkan support:

    • Real Time GPU signals are not yet available.
    • The Range Profiler is not supported.
    • Dynamic shader editing is not yet available.

    These will be added in a future release.

  • Added OpenGL Pixel History
  • Range Profiler improvements
  • Various bug fixes and performance improvements

Known Issues:

  • OpenGL Pixel History view might timeout for some applications with a large amount of draw calls.
  • If the Range Profiler view is closed while a selected event is being profiled, the host may crash. As a workaround, wait for the selected event to be profiled before closing the Range Profiler view. (58761)
  • The spin control in the histogram can incorrectly allow the min/max values to go out of range. This will be fixed in a future release. (55746)
  • The Shader Editor title bar may show the Fragment Shader or Vertex Shader rather than the Program name. (58147)
  • The timer function used for determining the CPU time for API calls is not accurate enough for shorter calls, resulting in '0' time values. This will be addressed in a future release. (58591)
  • HWPM Context Switching is not currently supported; this will be added in a future release.
  • If your application uses deprecated features/enums from OpenGL, and you try and generate C++ source code from a captured frame, you may encounter compiling issues. This is due to Tegra Graphics Debugger using updated include files from Khronos. If you point the build to older versions containing the deprecated features, it should compile without any issues. (40442)
  • Applications that use glDrawTransformFeedback will not see correct profiling data for those calls. Other draw calls in the frame should be correct. (35563)
  • If you encounter an issue where you are not able to capture, you should use the adb logcat command to see if there are any warning messages. The types of messages commonly seen are: 
    1. Unsupported functions (see Supported OpenGL Functions for a complete list)
    2. Use of OpenGL ES 1.0 (which is not supported)
    3. Out of memory issues (31527) 

Android Users Only

  • Due to a security change, some profiler features (such as real-time signals and the Range Profiler) may not be functional by default. This will be fixed in a future release. As a workaround, use the following steps prior to launching your application: 
    adb root
    adb shell chmod a+rw /dev/nvhost-prof-gpu
  • If you are seeing periodic failures connecting to a launched application, it could be because of a conflicting version of adb running on your host system. These can sometimes come from phone and tablet sync applets that run in the background, waking up periodically to check if a device is attached. Since they have their own (potentially older) version of adb in the install package, it can conflict with a newer one running from the NDK, installed by CodeWorks for Android and Tegra Graphics Debugger. Uninstalling the sync app should fix the conflict.

Linux Users Only

  • When using a target running Linux that uses ssh instead of adb, Tegra Graphics Debugger is unable to connect to the target device as a "root" user. This is because ssh config has forbidden root password authentication. If you check the contents of the target root fs /etc/ssh/sshd_config, you will find the following line:  
    PermitRootLogin without-password

    As a workaround, change the line to:

    PermitRootLogin yes

    Alternatively, you can log in with a different user account, and should not encounter this same error. (33646)
