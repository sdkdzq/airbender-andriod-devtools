Frame Scrubber

Understanding the Scrubber

For the sake of discussion when it comes to graphics debugging, it helps to note some common terminology.

  • An event is a single call to the API. It could be a draw call, or clear, or a less obvious call, like configuring API state.
  • An action is a subset of the event types. It can be a draw call, clear, blit, compute dispatch, or swap buffers. Actions are interesting since they explicitly change buffers, which may result in visual changes.

The default view will show the events in your application, in addition to any performance markers you have defined. Clicking the Add... button will open a dialog that allows you to select what type of range you want to add.

  • Pipeline — Actions that use the same pipeline.
  • Framebuffer — Actions that use the same Framebuffer.
  • User — A range defined by you on the fly. Use SHIFT + left-click and drag the Scrubber on the created "User" row to create a new range.

Scrubbing on the Host

Zooming In and Out

There are several ways to zoom in and out of the Scrubber in the Frame Debugger.

  1. Select an event range: hold down the SHIFT key and click-and-drag the mouse along the event timeline.
    The selected range appears as a dark gray area.
  2. Click inside the range.
    The event timeline zooms in, expanding the range to the borders of the timeline.

Alternatively, you can click-and-drag the either of the handles of the horizontal scroll bar.

You can also zoom in and out by holding the CTRL key while moving the scroll wheel on the mouse.

Scaling and Hierarchy

There are several ways to sort the events on the Scrubber.

  • Event ID — Unit Scale is the default view, which simply shows the actions and events on the timeline.
  • Sequence ID shows how the calls may have executed on the GPU in parallel.
    Note that Sequence ID is only used in APIs that allow for simultaneous execution, so it will look the same as Event ID ordering for APIs like OpenGL and OpenGL ES.
  • Event ID — CPU Time Scale displays the CPU activity and how much each event or action cost the CPU.
  • Event ID — GPU Time Scale displays the GPU activity and how much each event or action cost the GPU.
  • Event ID — X by CPU, Y by GPU displays the CPU time scale on a horizontal X-axis, and the GPU time scale on a vertical Y-axis.

View Hierarchy

From the Hierarchy drop-down, Queue Centric sorts the events by queue, while Thread Centric sorts the events by the thread.

Scrubbing on the Target

On a target, you can scrub through the events and actions, similar to the way you can on the host machine. There are 2 types of target scrubbing that are supported.

Tablet

Use one finger touch gestures to scrub through the scene. These include: 

  • Tap your finger on the Scrubber overlay and remove to set the current action.
  • Swipe with your finger over the Scrubber overlay to scrub through the scene, action by action.
  • Tap your finger on the left side of the screen, above the Scrubber overlay, to go back 1 action, right side to go forward.
  • Use 2 fingers on left side of the screen, above the Scrubber overlay, to go to first action, right to go to last action.

SHIELD Console

Using the Blake controller with the SHIELD console, you can do the following: 

  • DPad L to go back 1 action, DPad R to go forward. This includes key repeat.
  • Use the joystick to scrub variable speed (depending on the deflection of the stick).
  • X to go to first action.
  • B to go to last action.
